# Example New Skill Backlog Entry

- `title`: Add `quality-sonar`
- `change-type`: `new`
- `skill-id-or-name`: `quality-sonar`
- `problem-or-motivation`: capture Sonar-based quality checks as a reusable
  quality skill
- `reusability-rationale`: multiple repositories may want the same static
  analysis review behavior

  `quality-crap`, `gates-exit`
- `source-context-or-notes`: derived from issue body and local quality-gate
  setup
